MathDuel
=========

Краткое описание
-----------------

MathDuel — простая соревновательная игра для двух игроков, в которой участники решают математические примеры и соревнуются за большее количество правильных ответов. Игра генерирует вопросы разных уровней сложности и отображает варианты ответов; игроки выбирают ответ быстрее соперника.

Технологии
----------

- Jetpack Compose (UI)
- Material Design 3 (Material3)
- Koin (Dependency Injection)
- Kotlin Coroutines

Основная идея
-------------

Приложение предоставляет живую парную механику: два игрока видят один и тот же вопрос и должны как можно быстрее выбрать правильный вариант. Вопросы генерируются динамически и могут иметь разные уровни сложности (Easy / Medium / Hard). После выбора правильного ответа игрок получает очки, игра продолжается до заранее заданного числа раундов.

Ключевые компоненты
-------------------

- `MainActivity` — точка входа, собирает отображение приложения и передаёт зависимости в ViewModel.
- `StartGameScreen` — экран ввода имён игроков и выбора сложности.
- `GameScreen` — основной игровой интерфейс, показывает вопрос и панели игроков.
- `GameViewModel` — управляет состоянием игры, получает вопросы и обрабатывает ответы.
- `GameRepositoryImpl` — локальный генератор вопросов (логика генерации и вариантов ответов).

Инструкция по запуску
--------------

Требования:

- Android Studio (рекомендуемая версия — последняя стабильная).
- Установленный Android SDK и AVD (эмулятор) или подключённое Android-устройство.
- JDK 11 или новее.

Запуск через Android Studio:

1. Откройте проект в Android Studio (File → Open... → выберите корень проекта).
2. Подождите, пока Gradle синхронизируется и зависимости загрузятся.
3. Выберите устройство/эмулятор и нажмите Run (зелёная кнопка). Android Studio соберёт и установит приложение.

Запуск из командной строки (Windows PowerShell):

1. Откройте PowerShell и перейдите в корень проекта:

```powershell
Set-Location -Path "L:\MathDuel"
```

2. Собрать APK (debug):

```powershell
.\gradlew.bat assembleDebug
```

3. Установить и запустить на подключённом устройстве/эмуляторе:

```powershell
.\gradlew.bat installDebug
```

4. (Опционально) Установить конкретный APK вручную через ADB:

```powershell
adb install -r .\app\build\outputs\apk\debug\app-debug.apk
```

Советы при проблемах:

- Если Gradle не может скачать зависимости, проверьте интернет и настройки proxy в Android Studio.
- Если устройство не отображается, убедитесь, что включена отладка по USB и драйверы установлены (для Windows).
- При ошибках сборки просмотрите вывод Gradle в Android Studio (View → Tool Windows → Build).

Примечания
---------

- Генерация вопросов реализована локально в репозитории; при необходимости можно заменить на сетевой источник.
- Логика сложности и задержки подсветки ответов реализована в UI и ViewModel; при желании можно перенести дополнительную логику в слой домена.

Чистая архитектура
------------------

Проект организован по принципам чистой архитектуры с разделением на слои Presentation, Domain и Data:

- Presentation — UI (Jetpack Compose) и `GameViewModel`, отвечает за отображение и взаимодействие с пользователем.
- Domain — бизнес-логика и use case'ы (`GetQuestionUseCase`, `CheckAnswerUseCase`) и модели (`Question`, `Player`, `GameState`).
- Data — реализация источников данных (`GameRepositoryImpl`), генерация вопросов и DTO.

Такое разделение повышает модульность, облегчает тестирование и позволяет менять реализацию источников данных без изменения бизнес-логики.
